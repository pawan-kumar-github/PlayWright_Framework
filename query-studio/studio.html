      Query Reference
      
    

    

      
Model

        

          
TableLifecycle subfolder (STP, OTDTC, RT…)

          
RowOne iteration record from any file in that lifecycle

          
_TCTC key the row belongs to (TC1, TC2…) — virtual

          
_FILESource JSON filename — virtual, filterable

        

      

      
Statements

        
SELECT <cols|*> FROM <table|*> [WHERE <cond>]

UPDATE <table|*> SET col='val'[,...] [WHERE <cond>]

INSERT INTO <table> VALUES (TCID='TC10', ITERATION='1', col='val',...)

DELETE FROM <table|*> WHERE <cond>

ALTER TABLE <table|*> ADD COLUMN <col> DEFAULT '<val>'
ALTER TABLE <table|*> DROP COLUMN <col>
ALTER TABLE <table|*> RENAME COLUMN <old> TO <new>

DESCRIBE <table|*>

      

      
WHERE operators

        
=  !=  >  <  >=  <=  LIKE '%pattern%'
AND  OR   (left-to-right, can filter on _TC and _FILE too)

      

      
File routing

        

          
SELECT / DESCRIBEReads all files in the lifecycle, merged

          
UPDATE / DELETEWrites back to each row's source file

          
INSERTAppends to the file that already has the TCID, else first file

          
ALTERApplies to every file in the lifecycle
